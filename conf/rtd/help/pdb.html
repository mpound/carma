Help for the CARMA Project Database Information Window

General Tab

The Timestamp field displays the timestamp where this monitor data was updated. It should update every frame unless there is a problem in the carmaProjectDatabaseManagerHost application.

The online field is set to true every frame that the carmaProjectDatabaseManagerHost application is running.

The currentMethod and currentMethodStart fields show information about which CORBA method is running at the current instant, and how long it has been running. These fields are only updated for CORBA methods which access the database. All other methods are pure C++, and should be extremely fast.

Server Info Tab

This tab displays information about each server in the MongoDB replication dataset. The information displayed is identical to that produced by the mongo shell command rs.status(). It should be helpful in diagnosing any problems with replication to offsite locations.

The timestamp field displays the time when the information for this server was updated.

The name field displays the hostname and port number of one member of the MongoDB replica set.

The health field contains a value (0 or 1) which tells you whether this member of the MongoDB replica set is healthy.

The uptime field contains the number of seconds that this member of the MongoDB replica set has been running. This is the amount of time that the MongoDB server process has been running, not the host uptime.

The ping field contains the ping time in milliseconds, as determined by the MongoDB replica set peer you are querying from. In this case, it should always be the ping time from sdp.mmarray.org to the other peers. Lower is better.
