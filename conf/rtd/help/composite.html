Array Composite Window Help

This display gives the observers an overview of an entire subarray at a glance. The window is composed of 5 tabbed folders, one for each subarray, with the contents of each window tuned for that subarrays capabilities.

In the top panel are information about the state of the program controlling the subarray, the current project, source, tuning, weather, integration, and CARMA processes.

Subarray and Project Data:

  • Subarray - The name of this subarray
  • Running - True or false; if the CARMA program controlling this subarray is running (/opt/rt/bin/subarrayControl). This cell should be green under normal circumstances.
  • Initialized - True or false; if this subarray has been fully initialized by subarrayInit.py. This cell should be green under normal circumstances.
  • Track Mode - Mode of antenna tracking, on of RA/DEC, AZ/EL, MIXED, or NONE.
  • Project - The project name
  • ObsblockID - The fully-qualified observing block identification, project.obsblock[.subobsblock].trial

Source Data:

  • Source - The antenna tracking source name.
  • RA - J2000 Right Ascension of the source.
  • dRA(p.c.) - RA offset of the phase center.
  • DEC - J2000 Declination of the source.
  • dDEC(p.c.) - DEC offset of the phase center.
  • Src Vel. - The catalog velocity of the source (e.g. VLSR)
  • Doppler Src - The source used for Doppler tracking of the local oscillators. This may be different than the antenna tracking source.

Weather Data:

  • T(amb) - The outside ambient temperature in Celsius.
  • RH - The outside relative humidity in percent.
  • rho(H2O) - The current atmospheric water density in gram per cubic meter; under normal conditions, this is a number close to 1.
  • pwv(mH2O) - The derived precipitable water vapor column, in mm. In good weather this should be less than 10.

Azimuth/Elevation Plot

The circular plot on the right is a polar Az-El plot showing the source azimuth and elevation (yellow dot); the average direction the antennas are pointing (green triangle) the wind speed in mph and the direction from which the wind is blowing (white arrow).

Miscellaneous Data:

  • Integrating- Whether or not the correlator pipeline is integrating.
  • Int #- The integration number. This has the value of the framecount at the start of the integration. ( number of half-seconds since Jan 1, 2000 ).
  • Int Time- The requested integration time in seconds.
  • Record- The current integration record since the start of the integration. This number should count up as the integration proceeds.
  • of- The total number of integration records that will make up this integration; equal to the number of repeats requested in the integrate command.
  • Noise Source Enabled- Is the noise source on (true) or off (false)?
  • CARMA System Processes NOT Running- The number of array processes that have stopped or failed to start. If this number is not zero, check the RTD processes page and take appropriate action to restart the stopped process (e.g imrConsole restart). This cell should be green under normal circumstances.
  • Alarm Enable- Is the alarm system enabled? This cell has a value of ENABLED, TRIGGERED, or DISABLED. The last two are error conditions. This cell should be green under normal circumstances.
  • Alarm On- Is the alarm on? If so, you should hear it in the control room! This cell should be green under normal circumstances.

    • Antenna Table

    In the middle panel is some basic data about the antennas.

    • Drive State - Individual tracking state for this antenna.
    • ActAz - Actual azimuth in degrees
    • ActEl - Actual elevation in degrees.
    • AzSkyErr - Azimuth error on the sky in arcseconds.
    • ElErr - Elevation error in arcseconds.
    • dra - Right Ascension pointing offset in arcminutes
    • ddec - Declination pointing offset in arcminutes
    • RxState - Receiver state (e.g. GOOD, BAD).
    • Dsb - Double sideband receiver temperature for Band 1 in Kelvin.
    • Pad - Pad number to which this antenna is assigned.

    Subarray Membership Changes

    When membership in any subarray changes do to someone issuing addAntenna() or removeAntenna(), a highlighted message will appear asking you to Morph the display. Do this by selecting Array Composite off the Morph menu.
